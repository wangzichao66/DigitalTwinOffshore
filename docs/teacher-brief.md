# 教师与答辩说明

## 这个作品是什么

这是一个“基于数字孪生的海上能源场站多灾种智能仿真预警原型平台”，重点是把海面溢油和空中气云放进同一个三维孪生场景中统一展示，并基于简化规则输出预警和处置建议。

## 为什么不是工业级 CFD 系统

本科课程项目更适合做“原型平台”而不是“高精度事故后果软件”。原因很直接：

- 工业级 CFD 需要大量专业参数、网格划分和验证数据
- LNG/天然气后果分析涉及复杂热力学和边界条件
- 真正高精度模型计算量大，不适合课堂演示中的实时交互

所以本项目做了一个工程上更合理的取舍：

- 离线复杂模型不做
- 在线实时展示做强
- 用简化物理趋势替代高成本数值求解
- 把重点放在“统一场景、态势感知、联动预警、what-if 推演”

## 作品的亮点

### 1. 多灾种统一展示

不是只做溢油，也不是只做气体扩散，而是在一个界面里同时展示：

- 海面油膜扩散
- 空中气云扩散
- 风流浪工况变化
- 受影响目标
- 应急动作建议

### 2. 架构接近真实系统

虽然本轮没有做真实后端，但前端架构已经按真实系统分层：

- 数据接口层
- 仿真层
- 风险评估层
- 三维场景层
- 业务 UI 层

后续只要替换数据源，就可以向真实系统继续扩展。

### 3. 支持课堂演示和答辩

老师可以直观看到：

- 风速风向变化如何影响扩散
- `LNG` 与 `CNG` 的扩散行为差异
- 围油栏、撤离、航警的自动联动逻辑
- 面板指标、三维场景和传感器读数之间的关联

## 答辩时建议这样讲

1. 先说明选题背景：海上场站泄漏风险具有海空跨介质、多灾种耦合特征
2. 再说明作品定位：课程级数字孪生原型，不宣称工业级高精度求解
3. 然后展示三维场景与四大面板
4. 接着切换 `LNG/CNG` 和风流浪工况，展示动态变化
5. 最后强调系统价值：统一态势认知、演示推演、方案比选、教学训练

## 可以诚实说明的简化点

- 溢油采用简化粒子扩散模型
- 气云采用简化三维粒子近似模型
- 风险阈值采用规则引擎而非行业标定值
- 传感器数据为模拟遥测数据

这不是缺点，只要表达清楚“这是一个可扩展的系统原型”，这个取舍就是合理的。
